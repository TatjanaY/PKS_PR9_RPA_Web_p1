# **Кейс: «Веб-портал управления заявками на участие в IT-мероприятиях»**

---

## 📖 Ситуация

Компания **EventumTech** занимается организацией IT-мероприятий: хакатонов, митапов, онлайн-курсов и конференций.

Для регистрации участников используется веб-форма, через которую пользователи подают заявки на участие. В периоды активных регистраций сотрудники вручную проверяют и обрабатывают сотни заявок, что приводит к:

* задержкам обработки;
* ошибкам при вводе данных;
* перегрузке персонала.

Для оптимизации процесса было принято решение:

* разработать **учебный веб-портал регистрации заявок**;
* протестировать возможность автоматизации обработки заявок с помощью **RPA-робота (Puzzle RPA)**.

---

# **Задание 1. Разработка веб-приложения (детализированные требования)**

Необходимо разработать веб-приложение, имитирующее процесс регистрации заявок на участие в IT-мероприятиях.

---

## 🌐 Общая логика работы приложения

Приложение должно реализовывать следующий пользовательский сценарий:

1. Пользователь открывает страницу авторизации
2. Выполняет вход в систему
3. Переходит на страницу подачи заявки
4. Заполняет форму
5. Отправляет заявку
6. Видит подтверждение и добавленную запись в таблице

---

# 🔐 Страница авторизации

---

## 📝 Поля ввода

### `login`

**Назначение:**
Идентификация пользователя в системе

**Требования:**

* обязательное поле
* минимальная длина — 4 символа
* допускаются:

  * латинские буквы
  * цифры

---

### `password`

**Назначение:**
Аутентификация пользователя

**Требования:**

* обязательное поле
* минимальная длина — 6 символов

---

## 🔘 Кнопка

### `Sign In`

**Поведение:**

* не выполняет вход, если поля пустые
* при нажатии:

  * выполняется проверка данных
  * при ошибке отображается сообщение

---

## ⚠️ Валидация

### Ошибки:

* если поля пустые:

  ```
  Please fill in all fields
  ```

* если данные неверные:

  ```
  Invalid credentials
  ```

---

## ✅ Успешный вход

* выполняется переход на страницу заявки
* появляется основной интерфейс приложения

---

# 📝 Страница подачи заявки

---

## 📝 Поля ввода

---

### `Participant Name`

**Назначение:**
Имя участника мероприятия

**Требования:**

* обязательное поле
* только текст
* минимальная длина — 3 символа
* не допускаются цифры

---

### `Email`

**Назначение:**
Контактный email

**Требования:**

* обязательное поле
* должен соответствовать формату email

Пример корректного значения:

```
user@example.com
```

---

### `Event Name`

**Назначение:**
Название мероприятия

**Требования:**

* обязательное поле
* допускается текст и цифры
* минимальная длина — 3 символа

---

## 🎛 Выпадающий список

### `Participation Format`

**Назначение:**
Формат участия

**Значения:**

* `Online`
* `Offline`

**Требования:**

* обязательный выбор
* значение по умолчанию отсутствует

---

## ☑️ Чекбокс

### `Accept Terms`

**Назначение:**
Подтверждение согласия с условиями

**Требования:**

* обязательно должен быть установлен
* без него отправка невозможна

---

# ⚙️ Логика валидации формы

Форма считается **валидной**, если:

* заполнены все поля
* email соответствует формату
* выбран `Participation Format`
* установлен чекбокс `Accept Terms`

---

## 🔘 Кнопка

### `Submit Application`

---

## ⚙️ Управление состоянием (`disabled`)

Кнопка должна:

* иметь состояние `disabled = true`, если:

  * хотя бы одно поле пустое
  * email некорректен
  * не выбран формат участия
  * чекбокс не установлен

* иметь состояние `disabled = false`, если:

  * все условия выполнены

---

## ⚠️ Поведение при ошибках

При попытке отправки (если реализовано):

* подсветка некорректных полей
* вывод сообщений:

Примеры:

```
Invalid email format
```

```
Name must contain only letters
```

---

# 📤 Отправка формы

При нажатии `Submit Application`:

1. выполняется финальная проверка
2. создаётся новая запись
3. очищается форма
4. отображается модальное окно

---

## 🪟 Модальное окно

Текст:

```
Application submitted successfully
```

---

## ⚙️ Требования к поведению

* появляется с задержкой (1–2 секунды)
* автоматически исчезает через 2–3 секунды
  **или**
* закрывается кнопкой

# **Задание 2. Базовая автоматизация веб-приложения**

Необходимо разработать процесс автоматизации в **Puzzle RPA**, имитирующий действия пользователя в браузере.

---

## 🤖 Алгоритм работы робота

1. Создать сессию браузера.
2. Открыть веб-приложение.
3. Дождаться загрузки страницы авторизации.

---

4. Выполнить вход:

* ввести `login`
* ввести `password`
* нажать `Sign In`
* проверить успешность входа (например, по появлению формы)

---

5. Для каждой записи из входных данных:

### Ввод данных

* ввести `Participant Name`
* ввести `Email`
* ввести `Event Name`
* выбрать `Participation Format`
* установить `Accept Terms`

---

### Проверка состояния кнопки

* проверить, что кнопка `Submit Application` активна;
* если кнопка неактивна:

  * определить причину;
  * заполнить или исправить данные;

---

### Отправка формы

* нажать `Submit Application`

---

### Работа с модальным окном

* дождаться появления окна
* проверить текст:

  ```
  Application submitted successfully
  ```
* закрыть окно или дождаться его исчезновения

---

6. Повторить действия для всех записей (не менее 5).

---

7. Проверить, что записи появились в таблице.
